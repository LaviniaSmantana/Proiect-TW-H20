1. Description

We created this web application to help you manage your preferences for non-alcoholic beverage consumption.We also offer support for the management of related data: category, price, ingredients, images, restrictions, validity and others. Our system will provide support for shopping list activities, plus user administration.

2. Database

SoDrO uses a mySQL Database that stores the information about the users and the products.

The table used for the administration of the users is called: users and contains attributes such as:

  • id - every user has an unique id
  • username
  • email - the email that the user registered
  • password - the password that the user registered stored as a hash
  • cpassword - the confirmed password
  • image

The table used for the administration of the products is called products and it contains attributes such as:

  • id - every product has an unique id
  • name - name of the product
  • code - every product has an unique code
  • price
  • image - every product has a representative image
  • bought - it represent the amount of product that was added to the cart

4. Account

4.1. Login

If an user wants to access the website he/she has to login firstly. For that, the user has to provide the username and the password that he/she has chosen when he/she registered. If all these fields are completed, the username is valid and the password is correct, a verification is made in the database to see if there is an user with the username provided and if the password matches. If all these steps are passed, the user will enter the webpage.

4.2. Register

If an user doesn't have an account, he/ she has to register. For that, the user has to provide an email adress, an username, a password ,then has to confirm the password and upload an image as a profile picture optionally. If the email is valid and it isn't been used by another user, also if the confirmPassword field matches with the password and if the image size isn't too large, the user is created. A message is shown for this and if there is any error, with any of these fields it shows an error which specifies exactly which is the problem. The password is stored crypted in the database.

Product Visualisation

Our website provides the user a way to visualize several drinks from six different categories and to add his/her favourite drinks to the cart.

Data Export

Every way of data visualisation also has the option to export the data in different formats:

  • CSV - the data is written, containing the attributes selected in that specific view and their values
  • SVG - it saves the whole chart as a 2D image using XML
