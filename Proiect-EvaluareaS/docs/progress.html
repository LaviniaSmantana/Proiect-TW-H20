Web design

We started the project by creating the main elements such as navigation bar, the home page, the login and register pages, and a design for the products page. From that, we created the cart, the single product pages and the statistics page.

User Database

We continued by creating the users database and then proceeded to implement the login and register part. At this step we added login and register pages, such that if somebody wants to enter the webpage he/ she needs to create an account firstly.

Product Database

We continued by creating the products database and then proceeded to implement the shopping cart.

Download buttons

At this step we implemented the download buttons for exporting data in two formats: CSV, SVG.

RSS Data Flow

By using the products database and selecting those in descending order from the most bought items, we created a ranking that is available as RSS data flow.

Code Restructuring & Hardcoding Removal

For every page of product we used to have a different .css file, so we created just one for every type of product. Moreover, instead of the .html files, we transitioned to .php files.

Responsiveness

We have modified the width and the height values so that the page can also be properly seen on a tablet or smartphone.

Validate CSS & HTML

At this step we eliminated some warnings that we had with the CSS file and HTML pages.

Code Administration

For keeping the code updated and managing it we used Trello and Messenger. Every time one of us had some useful update for the project as a whole and was sure it was ready, we would post it on Trello. As a matter of fact, each of the chapters above represents, more or less, a commit one of us did.

Moreover, besides these specific commits, we also had code refactorization commits, that consist in making the code more readable.

Another important aspect of Trello Administration was updating the code. After one's commit, the other one of us had the responsibility to keep up with the latest changes by pulling them.
